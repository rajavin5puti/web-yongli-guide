# web-yongli-guide

一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定网站或域名，仅作为通用 HTML 页面存放与索引的容器。

## 项目简介

本仓库主要用于收集、整理和发布各类独立的 HTML 页面。这些页面可能包含示例、演示、工具界面或其他静态内容，各自独立运行，不依赖后端服务。仓库本身不提供任何特定功能或服务，仅作为页面存档与展示的载体。

## 目录说明

- `/pages` — 存放所有独立的 HTML 页面文件，每个页面为一个独立的子目录或直接以 HTML 文件形式存在。
- `/assets` — 存放页面共用的静态资源，如 CSS、JavaScript、图片等（可选）。
- `/docs` — 存放与页面使用或维护相关的文档说明（可选）。
- `index.html` — 仓库根目录下的索引页面，用于列出所有可访问的 HTML 页面入口。

## 页面归档说明

每个独立的 HTML 页面应满足以下基本要求：

- 页面应自包含，不依赖外部不可控资源（可引用仓库内资源）。
- 页面文件名或目录名应具备一定描述性，便于识别内容。
- 页面内部可包含必要的说明文字或使用指引。
- 页面不应包含任何恶意代码、跟踪脚本或违反法律法规的内容。

提交新页面时，请确保：

1. 将页面文件放入 `/pages` 目录下，或按类别创建子目录。
2. 更新根目录下的 `index.html` 索引文件，添加新页面的链接（如果存在）。
3. 如有必要，在 `/docs` 中添加对应说明文档。
